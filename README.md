# Morning Brief Agent

An autonomous daily brief covering financial markets, finance & economics, U.S.
politics, and international geopolitics — generated by the Claude Agent SDK and
emailed to you every morning at **8:00 AM Central** via GitHub Actions.

## Repo layout

```
your-repo/
├─ .github/workflows/morning-brief.yml   # schedule + runner (place the .yml here)
├─ morning-brief-agent-prompt.md         # the agent's instructions (system prompt)
├─ run_brief.py                          # generates the brief and emails it
├─ requirements.txt                      # Python dependencies
└─ briefs/                               # auto-created archive of dated briefs
```

## One-time setup (~5 minutes)

1. **Create a GitHub repo** and add the four files above. Put the workflow file at
   `.github/workflows/morning-brief.yml`. The `briefs/` folder is created automatically.

2. **Get an Anthropic API key** at https://platform.claude.com. Set a monthly budget
   on the key so a runaway run can't surprise you.

3. **Get a Resend API key** at https://resend.com (the free tier is plenty). Without
   verifying a domain, Resend lets you email **the address you signed up with** — which
   is perfect for sending a brief to yourself. To email any other address, verify a
   domain in the Resend dashboard.

4. **Add repository secrets:** in your repo, go to *Settings → Secrets and variables →
   Actions → New repository secret*, and add these four:
   - `ANTHROPIC_API_KEY`
   - `RESEND_API_KEY`
   - `EMAIL_FROM` — use `onboarding@resend.dev` to start, or `you@yourdomain.com` once verified
   - `EMAIL_TO` — your inbox (must match your Resend signup email until you verify a domain)

5. **Test it right now:** open the *Actions* tab → *Morning Brief* → *Run workflow*.
   You should get an email within a few minutes. Manual runs ignore the 8 AM gate, so
   you don't have to wait until morning to confirm it works.

## Schedule & timezone

The workflow triggers at 13:00 and 14:00 UTC and only proceeds when it is exactly
08:00 in `America/Chicago`. That gives you one brief at 8 AM Central year-round, with
daylight saving handled automatically — no manual clock changes twice a year.

## Notes

- **Cost:** each run uses Anthropic API credits (many web searches and fetches plus a
  long generation). To cut cost, change `model="opus"` to `model="sonnet"` in
  `run_brief.py`.
- **Keeping it alive:** GitHub auto-pauses scheduled workflows after ~60 days of repo
  inactivity. This setup commits each day's brief to `briefs/`, and that daily commit
  counts as activity — so it won't pause.
- **Prefer Gmail/SMTP over Resend?** Swap the `send_email()` function in `run_brief.py`
  for an `smtplib` version (about 10 lines). Ask and I'll provide it.
- **No fabrication:** the prompt forces the agent to retrieve and verify every article
  link before the brief is sent.
